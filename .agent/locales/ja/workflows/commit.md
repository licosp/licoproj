# Commit ワークフロー

## 目的
コミットされていない変更を識別し、論理的にグループ化（機能、修正、リファクタリングなど）し、明確でアトミックなコミットを作成します。

## ステップ

### 1. 調査
- `git status` と `git diff` を実行して変更を分析します。
- 関連する変更をグループ化します（例：機能またはバグ修正ごと）。

### 2. ステージングとコミット
各論理グループごとに:
1. **ステージング**: `git add <files>` (または部分的なステージングには `git add -p`)。
2. **コミット**: `git commit -m "<message>"`
   - **言語**: 英語のみ。
   - **フォーマット**: `<Type>: <Subject>` (例: `Feat: Add login page`)。
   - **タイプ**: `Feat`, `Fix`, `Refactor`, `Docs`, `Style`, `Test`, `Chore`。
   - **本文**: オプションの詳細または issue 参照（例: `Closes #12`）。

### 3. レビューとプッシュ
- `git log` で履歴を確認します。
- リモートにプッシュ: `git push origin main` (まずテストが通ることを確認)。
