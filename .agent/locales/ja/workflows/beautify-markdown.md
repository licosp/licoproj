# Markdown美化ワークフロー

## 目的
Markdownファイルを視覚的に魅力的で整理された、人間の可読性に最適化された文書に変換します。このワークフローは、機械での解析よりも見栄えと明瞭さを優先します。

## 手順

### 1. コンテンツ分析
- 対象のMarkdownファイルを読み取る
- 主要なテーマと論理的なグループを特定する
- 繰り返しパターンや密集したセクションを確認する

### 2. 視覚的階層の強化
視覚的改善を適用：

#### 絵文字とアイコンの追加
- セクションに絵文字を使用（📝, 🗂️, 🛠️, 🐛 など）
- 素早くスキャンできる視覚的ランドマークを作成

#### 構造の改善
- 主要セクションを区切るために水平線（`---`）を追加
- 適切な見出しレベルを使用（タイトルはH1、主セクションはH2、サブセクションはH3）
- スキャンしやすいように番号付きリストや箇条書きを作成

#### 強調の追加
- 重要な用語には **太字** を使用
- 微妙な強調には *斜体* を使用
- 技術用語やファイルパスには `コードブロック` を使用
- 注釈、警告、ハイライトには引用ブロック（`>`）を使用

### 3. コンテンツの整理
関連項目を論理的なカテゴリにグループ化：
- テーマ別セクションを作成（例：「ファイル管理」、「ツールとインストール」）
- コンテンツを時系列または重要度順に並べる
- 長い文書には目次を追加

### 4. 磨き上げと洗練
最終仕上げ：
- セクション間の一貫したスペーシングを確保
- 導入注釈やコンテキストブロックを追加
- 関連する場合はメタデータ（日付、著者、目的）を含める
- 意味を保持しつつ冗長性を削除

## ガイドライン

### 優先事項
1. **人間の可読性を第一に**: スキャンと理解を容易にする
2. **視覚的魅力**: フォーマットを使用して魅力的な文書を作成
3. **論理的な流れ**: コンテンツを思慮深くグループ化し順序付ける

### 気にする必要がないこと
- AIの解析効率
- 厳格な技術的フォーマットルール
- 冗長性（理解を助ける場合）

## 出力
読みやすく、ナビゲートしやすい、美しくフォーマットされたMarkdownファイル。
