# Beautify Markdown ワークフロー

## 目的
Markdownファイルを視覚的に魅力的で、よく整理された、人間にとって読みやすいドキュメントに変換します。このワークフローは、機械解析よりもプレゼンテーションと明確性を優先します。

## ステップ

### 1. コンテンツの分析
- 対象のMarkdownファイルを読む
- 主要なテーマと論理的なグループを特定する
- 繰り返しパターンや密集したセクションに注目する

### 2. 視覚的階層の強化
視覚的改善を適用します:

#### 絵文字とアイコンの追加
- 絵文字を使用してセクションをマークします（📝, 🗂️, 🛠️, 🐛, など）
- 素早くスキャンできる視覚的なランドマークを作成します

#### 構造の改善
- 主要なセクションを区切るために水平線（`---`）を追加します
- 適切な見出しレベルを使用します（タイトルにH1、主要セクションにH2、サブセクションにH3）
- スキャンしやすくするために番号付きリストまたは箇条書きリストを作成します

#### 強調の追加
- 重要な用語には**太字**を使用します
- 微妙な強調には*斜体*を使用します
- 技術用語やファイルパスには`コードブロック`を使用します
- 注記、警告、またはハイライトには引用（`>`）を使用します

### 3. コンテンツの整理
関連項目を論理的なカテゴリにグループ化します:
- テーマ別セクションを作成します（例：「ファイル管理」、「ツールとインストール」）
- コンテンツを時系列順または重要度順に並べます
- 長いドキュメントには目次を追加します

### 4. 磨きと洗練
仕上げ:
- セクション間の一貫したスペーシングを確保します
- 導入注記やコンテキストブロックを追加します
- 関連する場合はメタデータ（日付、著者、目的）を含めます
- 意味を保持しながら冗長性を削除します

## ガイドライン

### 優先事項
1. **人間の可読性最優先**: スキャンと理解を容易にします
2. **視覚的魅力**: フォーマットを使用して魅力的なドキュメントを作成します
3. **論理的な流れ**: コンテンツを慎重にグループ化し並べます

### 気にしなくて良いこと
- AIの解析効率
- 厳格な技術的フォーマットルール
- 冗長性（理解を助ける場合）

## 出力
読みやすく、ナビゲートしやすい、美しくフォーマットされたMarkdownファイル。
