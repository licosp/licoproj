# ローカライゼーション標準

## 目的
多言語ドキュメントを維持するための翻訳とローカライゼーションの原則を定義します。

## 翻訳ガイドライン

### 技術用語
- **技術的正確性の保持**: 技術用語（例：`commit`、`workflow`、`git`、`markdown`）は明確性のためカタカナまたは英語のまま保持します
- **一貫性**: すべての翻訳ドキュメントで一貫した用語を使用します
- **コンテキスト対応**: コンテキストと対象読者に基づいて適切な翻訳を選択します

### 構造の保持
- **YAMLフロントマター**: フロントマターのメタデータは翻訳せずに保持します
- **見出し**: 見出しの階層とフォーマットを維持します
- **コードブロック**: コードスニペット、ファイルパス、コマンド例は変更せずに保持します
- **リンク**: ファイルリンクとURLは元の形式のまま保持します

### トーンとスタイル

#### 英語から日本語へ (EN → JA)
- **フォーマルな日本語**: プロフェッショナルなドキュメントには丁寧な日本語（です・ます調）を使用します
- **技術用語**: 技術用語はカタカナまたは英語のまま保持します（例：`コミット`、`ワークフロー`、または `commit`、`workflow`）
- **明確性**: 直訳よりも明確なコミュニケーションを優先します
- **自然な流れ**: 翻訳が日本語として自然に読めるようにします

#### 日本語から英語へ (JA → EN)
- **プロフェッショナルなトーン**: 技術ドキュメントに適した明確でプロフェッショナルな英語を使用します
- **技術用語**: すでに英語またはカタカナになっている技術用語は元の形式のまま保持します
- **慣用的な英語**: 逐語訳ではなく、自然な英語表現に翻訳します
- **簡潔性**: 完全性を維持しながら簡潔な英語表現を優先します

## 品質保証
- **正確性チェック**: 翻訳が元の意味と意図を保持していることを確認します
- **フォーマット検証**: Markdownフォーマットがそのまま保持されていることを確認します
- **一貫性レビュー**: 関連ドキュメント全体で用語の一貫性を確認します
