# コード品質標準

## 品質原則
- **可読性**: 巧妙さよりもコードの明確性を優先します
- **保守性**: 将来の開発者（ユーザーを含む）が簡単に理解できるコードを書きます
- **エラーハンドリング**: エッジケースを予測し、エラーを適切に処理します
- **テストマインドセット**: テスト可能性を考慮し、重要な機能のテストを提案します

## ツールの統合
- **Linter** (例：Python用のruff): よりクリーンなコードのためにlinterの提案に従います
- **型チェッカー** (例：Python用のmypy): コード理解を高めるために型ヒントを活用します
- **Language Server** (例：Pylance): IDEのフィードバックと診断を活用します
- これらのツールを使用して、より理解しやすく保守しやすいコードを提案します
