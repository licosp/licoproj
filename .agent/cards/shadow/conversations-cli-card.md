## Agent Observations

### Agate (2026-03-13)

**Script Specification: JSON to JSONL Converter (`json_to_jsonl.py`)**

- **Purpose**: To convert massive, single-file JSON logs (which bloat the Git repository due to line-based diff limitations) into date-partitioned JSONL (JSON Lines) format. This allows Git to track only incremental daily appends.
- **Arguments**:
  1. `input_json_path`: Path to the raw, monolithic `.json` file generated by Gemini CLI.
  2. `output_root_dir`: The root directory where partitioned `.jsonl` files will be saved (e.g., `.repos/.licoshdw/conversations_cli/identifiers/<id>/`).
- **Processing Logic**:
  - Parse the `messages` array from the input JSON.
  - Extract the date (`YYYY/MM/DD`) from each message's `timestamp`.
  - Minify the message object (remove all formatting spaces and newlines using `separators=(',', ':')` in Python).
  - Append the minified JSON string to `output_root_dir/YYYY/MM/DD/log.jsonl`. This groups messages by day, creating a new file only when the date changes.

### Agate (2026-02-05)

- **Purpose**: Backup of raw system logs (JSON) generated by Gemini CLI. These are essential for session resumption (`--resume`) and reconstructing the exact context window.
- **Location**: `.agent/.internal/.shadow/conversations_cli/`
- **Distinction**:
  - `[Conversations]`: Human-readable Markdown logs (The Story).
  - `[Conversations-CLI]`: Machine-readable JSON logs (The Data).
